# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.0.x | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, send a description of the vulnerability to the project maintainers via private communication. Include:

1. A description of the vulnerability and its potential impact.
2. Steps to reproduce the issue.
3. Any suggested mitigations or fixes.

We will acknowledge receipt within 48 hours and provide a timeline for resolution.

## Scope

This is a reference/educational project. Security considerations are documented for best-practice purposes. See `docs/SECURITY.md` for application-level security design notes.
